Input

Inputs enable the user to interact with and input data. Use when the application requires long-form content from the user.

Design and Behavior

Inputs can be text inputs, area inputs and tag inputs. Using one on another depends on the length of the data the user can introduce. As well depends on the type of data in case of the input tags. Text inputs are a free form input where the user can type anything. Text inputs are for short amounts of data, while area inputs are for long amounts of text. We should anticipate which one is the most suitable option for each case. The user in area inputs can make the area bigger dragging the input area from the bottom right.

Usage and Best Practices

Inputs have many states, and can have specific validation rules fot the introduced data. Therefore if the validation rules are not met an error message will appear. The user should allways know if specific rules are applied to that input that’s why we should put clear hints about what the user should introduce.

By default, the inputs have a label and a placeholder text that shows or an example of the text that it should be introduced or a call to action text. The label should be a clear statement of what the user is asked to respond on the input field.

When the user clicks on an input, the on focus state starts, the input gets blue and a blinking cursor appears at the beggining of the input so the user know is there. While the user doesn’t write anything the placeholder text remains. If the user starts to write, the placeholder disappears. If the user deletes the content, the placeholder appears again.

If the input has validation rules, they should be clearly explained in the hint text that area which appears immediately under the input and never disappears. When the user writes on this kind of inputs, an error message should appear only when the validation rules are not met and the user clicks outside the input. In that case an error message appears explaining why the validation has failed immediately under the hint text.
